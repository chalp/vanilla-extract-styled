# Правила написания кода для пакета `@chalp/vanilla-extract-styled`

## TypeScript

- Строго **TypeScript**, избегать `any`. Если тип неизвестен — использовать `unknown` и уточнять через type guard.
- Следовать **Airbnb JavaScript/TypeScript Style Guide**.
- Использовать `enum` для наборов констант (подтверждено практикой: `ButtonSize`, `Placement`, `TooltipType`, `PopoverPlacement` и т.д.).

---

## ESLint

- Конфиг: `eslint.config.mjs`, extends `@chalp/eslint-airbnb`.
- Все изменения проходят **ESLint** без ошибок и новых предупреждений.
- `tsc --noEmit` — без ошибок компиляции.
- Проверять перед коммитом.
- Не все правила безусловны — если правило избыточно для конкретного случая, спросить у пользователя про `// eslint-disable-next-line`.

---

## JSX / Обработчики событий

- Не передавать инлайн-функции с логикой в JSX.
- Логика в обработчиках — в именованные `handle`-функции.

---

## Общие принципы

- Минимизировать вложенность, использовать ранний выход.
- Компоненты-обёртки (`memo`) для оптимизации рендера.
- Публичный API — через `index.ts` модуля.
- Не использовать «магические числа» — выносить в именованные константы с осмысленным названием.

---

## Тестирование

- Фреймворк: **Vitest 4**.
- Тесты размещаются рядом с компонентом (`*.test.ts`, `*.test.tsx`) или в каталоге `__tests__/` внутри модуля.
- Покрывать: рендер, пропсы, обработчики событий, краевые случаи.
